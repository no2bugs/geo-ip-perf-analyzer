Help & Usage

Overview

GeoIP Latency Analyzer performs mass latency testing against a list of domains/IPs, resolves their geographic location using GeoLite2 databases, and optionally runs VPN throughput tests via OpenVPN.

Quick Start

  1. Add target domains/IPs using the Servers List button (one per line).
  2. Configure Pings per IP, Timeout, and Workers as needed.
  3. Click Start Scan to begin latency testing.
  4. Optionally select rows and run VPN Speedtest to measure download/upload via VPN.

Header Buttons

☰ Servers List
Open the server list editor. Enter one domain or IP address per line. Duplicates and blank lines are removed automatically on save.
↻ OVPN Configs
Upload a .zip file containing OpenVPN configuration files (e.g. the NordVPN ovpn.zip bundle from downloads.nordcdn.com). Only UDP .ovpn files are extracted. These configs are used for VPN speedtests. Hover to see the current config count and last update date.
↻ GeoLite2 DBs
Download the latest GeoLite2-City and GeoLite2-Country databases from GitHub (P3TERX/GeoLite.mmdb). Hover to see local DB date vs. latest available release.

Scan Controls

Pings per IP
Number of ICMP pings sent to each target. Higher values give more accurate averages but take longer.
Timeout (ms)
Maximum time to wait for a ping response before marking the target as unreachable.
Workers
Number of concurrent threads used for scanning. More workers = faster scans but higher system load.
Start / Stop Scan
Begin or abort a latency scan. Progress is shown in the bar below.
Select All / VPN Speedtest
Select result rows, then run a VPN speedtest. The app connects through each server's matching .ovpn config and measures throughput via speedtest-cli.

Results Table

Click any column header to sort. Use the search box to filter by domain, country, or city. The Refresh Data button reloads results.json from disk without running a new scan.

Latency is color-coded: green < 50ms, yellow < 150ms, red ≥ 150ms.
